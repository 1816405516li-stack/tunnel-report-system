# 项目结构说明

本项目按“页面交互、核心数据处理、Excel 生成器、固定资料库、临时工作区、正式输出”分层。

## 页面层

`pages/` 只负责网页交互，例如上传文件、选择月份、预览清洗结果、触发生成任务。

页面层不直接写 Excel，也不直接保存业务规则。

## 核心层

`core/` 负责把来源文件整理成稳定的内部数据。

推荐顺序：

1. `importers.py` 读取来源文件。
2. `cleaners.py` 清理原始数据。
3. `normalizers.py` 统一字段。
4. `validators.py` 校验数据。
5. `splitters.py` 拆分隧道和报表数据。
6. `calculators.py` 计算汇总数据。
7. `file_manager.py` 管理运行过程文件。

## 生成层

`generators/` 只负责把标准数据写入 Excel 模板。

每一种成果文件单独一个模块，避免总月报、单隧道月报、巡查表、故障记录单、电费表互相缠在一起。

## 资料库

`resources/` 保存长期重复使用的材料。

- `templates/`：Excel 模板。
- `references/`：依据文件、说明文件。
- `rules/`：规则文件。
- `mappings/`：字段映射、隧道名称映射、编码映射。

## 工作区和输出区

`workspace/` 保存每次运行的临时文件。

`outputs/` 保存正式成果文件。

两者分开，后续更容易清理临时文件，也更容易备份正式成果。
