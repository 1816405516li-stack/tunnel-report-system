# 数据流程草案

当前只是流程草案，后续实现时再逐步细化。

## 故障月报链路

1. 上传初始故障单 Excel。
2. 读取原始故障数据。
3. 清洗空行、日期、设备名称、故障描述和处置措施。
4. 标准化隧道名称、桩号、故障时间、修复时间、设备数量和更换数量。
5. 校验必填字段和异常数据。
6. 生成清洗后的月度标准数据。
7. 按总月报、单独隧道月报、日常巡查、经常性检查、故障记录单分别生成 Excel。

## 电费链路

1. 上传隧道账单表。
2. 上传往月电费表。
3. 读取当月电费数据和上月模板数据。
4. 清洗户号、电量、电费、功率因数等字段。
5. 校验当月数据和上月数据的对应关系。
6. 生成当月电费表。

## 固定资料链路

1. 从 `resources/templates/` 读取模板。
2. 从 `resources/rules/` 读取长期规则。
3. 从 `resources/mappings/` 读取字段、隧道名称、编码等映射。
4. 从 `resources/references/` 读取依据文件。
