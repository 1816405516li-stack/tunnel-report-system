# 数据流程

## 故障月报链路

1. 上传设备维修单 Excel。
2. 读取原始故障记录。
3. 清洗空行、日期、设备名称、故障描述和处置措施。
4. 标准化隧道名称、桩号、故障时间、修复时间、设备数量和更换数量。
5. 校验必填字段和异常数据。
6. 生成月度标准数据。
7. 按总月报、单隧道月报、日常巡查表、经常性检查表和故障记录单生成 Excel。

## 固定资料链路

1. 从 `resources/templates/monthly/` 读取长期复用模板。
2. 从 `resources/rules/` 读取报表规则。
3. 从 `resources/mappings/` 读取隧道、户号等映射。

## 本地运行数据

`workspace/` 和 `outputs/` 只保存本机运行过程和生成成果，不进入 GitHub。
